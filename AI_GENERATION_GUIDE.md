# AI 生成数据格式指南

本文件约定 2D×N 多层知识图谱展示软件所需的数据格式。任何 AI 工具在生成、补充或修改项目数据时，都应遵循本指南。

## 一、文件格式

- 主数据文件使用 **YAML**（推荐）或 **JSON**。
- 文件编码为 UTF-8。
- 文件后缀建议为 `.yaml` 或 `.yml`。

## 二、顶层结构

```yaml
project: "项目名称"      # 字符串，必填
version: "1.0"         # 字符串，必填
layers: []             # 图层数组，必填
crossEdges: []         # 跨图层连线数组，可选，默认为空
```

## 三、图层（Layer）

每个 `layer` 代表一个独立的 2D 流程图或思维导图。

```yaml
layers:
  - id: "layer-1"                # 图层唯一 ID，必填，全局唯一
    name: "战略目标层"            # 图层名称，必填
    description: "..."           # 图层描述，可选
    color: "#dbeafe"             # 图层主题色，可选（HTML 颜色值）
    x: 50                         # 图层原点 X，可选，默认 0
    y: 50                         # 图层原点 Y，可选，默认 0
    nodes: []                     # 指标节点数组，必填
    edges: []                     # 层内连线数组，必填
```

## 四、指标节点（Node）

```yaml
nodes:
  - id: "node-1-1"               # 指标唯一 ID，必填，全局唯一
    name: "提高客户满意度"        # 指标显示名称，必填
    type: "indicator"            # 指标类型，可选，默认 indicator
    x: 100                        # 相对图层原点的 X 坐标，可选，默认 0
    y: 80                         # 相对图层原点的 Y 坐标，可选，默认 0
```

> **重要**：指标使用全局唯一 ID 标识，`name` 仅用于展示。不同图层允许出现同名指标，但 ID 必须不同。

## 五、连线（Edge）

连线既可以在 `layers[i].edges` 中定义（同层内关系），也可以在 `crossEdges` 中定义（跨层关系）。

```yaml
edges:
  - id: "edge-1-1"               # 连线唯一 ID，必填
    source: "node-1-1"           # 起点指标 ID，必填
    target: "node-1-2"           # 终点指标 ID，必填
    direction: "forward"         # 箭头方向：forward / backward / bidirectional
    type: "drives"               # 连线类型，自由字符串
    brief: "满意度提升 → 投诉减少"    # 简要表述，点击高亮时显示在连线上
    detail: "客户满意度每提高 1 分..."  # 复杂表述，点击连线后弹窗展示
    table: "| 指标 | 数值 |\n|---|---|\n| ... | ... |"   # 关联表格，Markdown 表格格式
```

### 5.1 方向说明

- `forward`：箭头从 `source` 指向 `target`。
- `backward`：箭头从 `target` 指向 `source`。
- `bidirectional`：双向箭头。

### 5.2 连线不唯一

两个指标之间**允许存在多条连线**，只要 `id` 不同即可。例如：

```yaml
- id: "edge-a"
  source: "node-1"
  target: "node-2"
  direction: "forward"
  brief: "A 驱动 B"
- id: "edge-b"
  source: "node-1"
  target: "node-2"
  direction: "backward"
  brief: "B 反哺 A"
```

## 六、跨图层连线

跨图层连线统一放在 `crossEdges` 中，`source` 和 `target` 只需是任意两个指标节点的 ID，不要求在同一图层：

```yaml
crossEdges:
  - id: "cross-1"
    source: "node-1-1"
    target: "node-2-3"
    direction: "forward"
    type: "influences"
    brief: "满意度驱动售后响应"
    detail: "..."
    table: "..."
```

## 七、数据校验规则

1. 所有 ID（项目内）必须全局唯一：图层 ID、指标 ID、连线 ID 互不重复。
2. `source` 和 `target` 必须引用已存在的指标 ID。
3. `direction` 必须是 `forward`、`backward`、`bidirectional` 之一。
4. `x`、`y` 应为数值类型。
5. `table` 如使用 Markdown 表格，应包含表头分隔行 `|---|---|`。

## 八、AI 生成建议

- 先生成 `layers` 列表，明确每一层的业务含义。
- 为每层生成 `nodes` 时，确保 ID 有清晰前缀（如 `layer-1` 的节点用 `node-1-x`）。
- 同层关系放入 `edges`，跨层关系放入 `crossEdges`。
- 为每条连线撰写 `brief` 和 `detail`，`brief` 控制在 20 字以内，`detail` 可展开说明。
- 如有关联数据，用 `table` 字段以 Markdown 表格形式补充。

## 九、示例模板

```yaml
project: "新项目名称"
version: "1.0"
layers:
  - id: "layer-1"
    name: "第一层"
    color: "#dbeafe"
    x: 50
    y: 50
    nodes:
      - id: "node-1-1"
        name: "指标 A"
        type: "indicator"
        x: 100
        y: 100
    edges: []
crossEdges: []
```

## 十、Markdown 导入格式

除 YAML/JSON 外，系统也支持导入 `.md` / `.mmd` 文件。AI 在生成可被导入的 Markdown 时，应使用下面的**结构化格式**。该格式会被优先识别；如果不符合结构化格式，系统会回退到普通大纲或 Mermaid 流程图解析。

### 10.1 文件基本约定

- 文件后缀使用 `.md` 或 `.mmd`。
- 编码为 UTF-8。
- 每个 `# 图层：<名称>` 开始一个新图层，直到下一个图层标题或 `# 跨层关系` 结束。
- 节点名称在同一图层内应保持唯一，以便跨层关系正确匹配。

### 10.2 图层与节点

```markdown
# 图层：战略目标层

## 节点：提高客户满意度
- 简要表述：客户满意度提升
- 复杂表述：通过产品和服务改进，使客户满意度持续上升。
- 节点类型：indicator
- 关联表格：
  | 维度 | 目标值 |
  |---|---|
  | NPS | 50 |

### 减少客户投诉
- 简要表述：投诉量下降

### 提升复购率
- 简要表述：复购率提升

# 图层：执行措施层

## 优化售后流程
- 简要表述：缩短响应时间
```

解析规则：

- `# 图层：<名称>`（1~6 级标题均可）定义一个图层。
- `## 节点：<名称>` 或 `## <名称>`（2~6 级标题均可）定义一个节点。
- 标题的级数决定层级关系：级数更大的节点会成为离它最近、且级数更小的前序节点的子节点，并生成一条从父节点指向它的边。
- 如果同级或更高级标题出现，则先关闭之前的兄弟/子层级。

### 10.3 节点元数据

节点标题下方的列表项用于填充该节点的属性：

| 列表项前缀 | 作用 | 对应数据 |
|---|---|---|
| `- 简要表述：` | 作为从父节点指向该节点的边的 `brief` | 层内边 `brief` |
| `- 复杂表述：` | 作为层内边的 `detail` | 层内边 `detail` |
| `- 节点类型：` | 节点类型，当前默认 `indicator` | 节点 `type` |
| `- 节点备注：` | 保留字段，暂不写入模型 | — |
| `- 关联表格：` | 后续以 `\|` 开头的行会被收集为 Markdown 表格 | 层内边 `table` |

> 关联表格必须紧跟在 `- 关联表格：` 之后，中间不能插入其他非表格内容。

### 10.4 跨层关系

在任意位置使用 `# 跨层关系` 标题，后跟一个 Markdown 表格即可声明跨层边：

```markdown
# 跨层关系

| 源图层 | 源节点 | 目标图层 | 目标节点 | 方向 | 类型 | 简要表述 | 复杂表述 | 关联表格 |
|---|---|---|---|---|---|---|---|---|
| 战略目标层 | 提高客户满意度 | 执行措施层 | 优化售后流程 | forward | influence | 满意度驱动售后 | 客户满意度提升促使售后流程优化 | \| 指标 \| 数值 \| \n \| --- \| --- \| \n \| ... \| ... \| |
```

表格列说明：

1. **源图层**：源节点所在图层的 `name`。
2. **源节点**：源节点的 `name`。
3. **目标图层**：目标节点所在图层的 `name`。
4. **目标节点**：目标节点的 `name`。
5. **方向**：`forward` / `backward` / `bidirectional`。
6. **类型**：边的类型，自由字符串，如 `influence`、`feedback`。
7. **简要表述**：跨层边 `brief`。
8. **复杂表述**：跨层边 `detail`。
9. **关联表格**：跨层边 `table`，可留空。

如果源/目标节点在对应图层中不存在，导入时会自动创建同名节点。

### 10.5 完整示例

```markdown
# 图层：战略目标层

## 节点：提高客户满意度
- 简要表述：客户满意度提升
- 复杂表述：通过提升产品和服务质量，使客户满意度达到行业领先水平。
- 关联表格：
  | 维度 | 目标值 | 当前值 |
  |---|---|---|
  | NPS | 50 | 38 |
  | 复购率 | 60% | 45% |

### 减少客户投诉
- 简要表述：投诉量下降

### 提升复购率
- 简要表述：复购率提升

# 图层：执行措施层

## 节点：优化售后流程
- 简要表述：缩短响应时间
- 复杂表述：建立 7×24 小时响应机制，平均响应时间缩短至 2 小时内。

### 建立客户回访机制
- 简要表述：主动回访

# 跨层关系

| 源图层 | 源节点 | 目标图层 | 目标节点 | 方向 | 类型 | 简要表述 | 复杂表述 |
|---|---|---|---|---|---|---|---|
| 战略目标层 | 提高客户满意度 | 执行措施层 | 优化售后流程 | forward | influence | 满意度驱动售后 | 高满意度需要高效售后支撑 |
```

上述 Markdown 导入后将生成两个图层，并在两个同名节点之间建立一条跨层 `forward` 边。
